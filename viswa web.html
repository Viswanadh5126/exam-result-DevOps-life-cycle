Plan

Define the portal requirements:
• Hall ticket input box
• Button to check results
• Result display area with marks, grade, and status.

Code

HTML creates the structure:
• <input> for hall ticket
• <button> to trigger checkResult()
• CSS styles for clean UI
• JavaScript for logic.

Build

JavaScript creates the studentData object:
• Keys = Hall Ticket numbers
• Values = Student name, subjects, marks, pass/fail status.

Test

checkResult() checks:
• Empty hall ticket alert
• Valid ticket lookup in studentData
• Correct grade calculation from percentage.

Release

Result Box displays:
• Name
• Subject-wise marks table
• Total marks, percentage, grade
• Pass/Fail status.

Deploy

JavaScript event listener enables Enter key to trigger results without clicking the button.

Operate

Portal in use:
• Students enter hall ticket
• View instant results with marks and status.

Monitor

Plan improvements:
• Integrate live database
• Secure input validation
• Add printing/downloading option.
